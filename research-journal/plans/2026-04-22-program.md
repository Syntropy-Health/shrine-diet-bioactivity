# Diet-OS Program Plan

> **For agentic workers:** This is a program-level roadmap of seven subsystems. Each subsystem gets its own detailed TDD plan under `research-journal/plans/`. Execute subsystems in dependency order; within a subsystem, execute tasks in order.

**Goal:** Ship two peer-reviewed publications (primary β + companion γ) anchored on a provenance-grounded agentic harness over a unified diet-bioactivity-TCM knowledge graph.

**Architecture:** Seven independent subsystems (A–G) with explicit dependencies. Each produces working, testable software (or, for G, a submittable draft) on its own.

**Tech Stack:** Python 3.10+, LightRAG (HKUDS), Neo4j Aura, better-sqlite3, pytest, Bayesian Optimization (scikit-optimize or Ax), OpenAI / Anthropic SDK.

## The seven subsystems

### Subsystem A — Data moat completion

**Status:** Plan written (`plans/2026-04-22-subsystem-a-data-moat.md`)
**Depends on:** Phase B Neo4j (user-blocked: password rotation + Infisical)
**Produces:** Full-breadth KG in Aura with SymMap + HERB 2.0 + food bridge + HDI-Safe 50, re-ingested at full scale and queryable via LightRAG `/query`. Ingestion report with entity/edge counts checked into `shared/ingestion-snapshot.md`.
**Key outputs:**
- `data_local/herbal_botanicals.db` enriched with HDI + SymMap symptoms
- `shared/hdi_safe_50.json` curated interaction file
- `shared/symptom_crosswalk.json` Duke↔SymMap alignment
- Full-scale LightRAG KG in Aura (subsample unpinned)
**Estimated effort:** 3–4 weeks

### Subsystem B — Stage 1 Clinical Intake Agent

**Status:** Plan pending
**Depends on:** LLM provider (Anthropic/OpenAI) + schema validator library
**Produces:** Python service that transforms raw user query into `structured_intake` JSON conforming to OPQRST + SOCRATES + NCP/ADIME schema; clarification loop bounded at 2 rounds; red-flag classifier.
**Key outputs:**
- `shrine-diet-bioactivity/agents/intake.py`
- `shrine-diet-bioactivity/agents/intake_schema.py` (Pydantic)
- `tests/agents/test_intake.py` (unit + integration)
**Estimated effort:** 1.5–2 weeks

### Subsystem C — Stage 2 KG retrieval + typed-chain extractor

**Status:** Plan pending
**Depends on:** A, LightRAG `/query` endpoint
**Produces:** Service that maps `structured_intake` → LightRAG dual-level keywords → hybrid-mode query → typed-chain extractor → candidate_chains[] with source_ids, weights, evidence-tier labels.
**Key outputs:**
- `shrine-diet-bioactivity/agents/retrieval.py`
- `shrine-diet-bioactivity/agents/chain_extractor.py`
- `tests/agents/test_retrieval.py`
**Estimated effort:** 2 weeks

### Subsystem D — Stage 3 Adaptive expert panel

**Status:** Plan pending
**Depends on:** B, C
**Produces:** MDAgents-style triage + 6-role agents with MedAgents-style debate-consensus; dissenting-minority report generator.
**Key outputs:**
- `shrine-diet-bioactivity/agents/panel/triage.py`
- `shrine-diet-bioactivity/agents/panel/roles/{dietitian,pharmacologist,tcm,research_scientist,safety,defer}.py`
- `shrine-diet-bioactivity/agents/panel/debate.py`
- `tests/agents/panel/`
**Estimated effort:** 3 weeks

### Subsystem E — Stage 4 Calibrator + provenance formatter

**Status:** Plan pending
**Depends on:** D
**Produces:** Bayesian linear fusion with BayesOpt-tuned weights, Platt/isotonic calibration, Dempster-Shafer ablation baseline, provenance-chain JSON formatter with Cypher faithfulness verification.
**Key outputs:**
- `shrine-diet-bioactivity/agents/calibrator.py`
- `shrine-diet-bioactivity/agents/provenance.py`
- `tests/agents/test_calibrator.py` (incl. calibration property tests)
**Estimated effort:** 2 weeks

### Subsystem F — DietBench-Clinical + evaluation harness

**Status:** Plan pending
**Depends on:** B, C, D, E
**Produces:** 200-scenario benchmark with dietitian + pharmacist gold standards, stratified splits, leakage controls, baseline implementations (GPT-4 zero-shot, LightRAG naive, Yang 2025 two-agent, MedAgents, MDAgents), metrics module (ranking, calibration ECE, HDI recall, provenance faithfulness), reproducible eval runner.
**Key outputs:**
- `research-journal/shared/datasets/dietbench_clinical_v1.json`
- `research-journal/shared/datasets/splits_seed42.json`
- `shrine-diet-bioactivity/eval/{runner,baselines,metrics}.py`
- `research-journal/shared/results/` (JSON result dumps per run)
**Estimated effort:** 4–5 weeks (half spent on annotation + IAA)

## Cross-cutting concerns

- **Credentials:** all secrets via Infisical "SyntropyHealth App". No literals in any tracked file.
- **TDD:** every subsystem plan uses RED → GREEN → REFACTOR → COMMIT per task.
- **Coverage:** ≥ 80% per subsystem's test module.
- **Pinning:** LLM model versions pinned + date-stamped per eval run; random seeds pinned for splits.
- **Review:** `code-reviewer` agent after each subsystem's implementation is complete; `pr-test-analyzer` before marking evaluation complete.
- **Plans written iteratively:** B–G plans get written as A nears completion.
